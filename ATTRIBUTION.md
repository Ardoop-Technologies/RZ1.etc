# ATTRIBUTION.md – Recognition Framework

**Ethical AI MY – Contributor Recognition and Citation Standards**

---

## Overview

This document establishes the framework for recognizing and attributing contributions to Ethical AI MY. It provides guidance on how contributors should be acknowledged, how sources should be cited, and how intellectual property is recognized.

---

## Contributor Recognition

### Types of Contributions

Contributions include but are not limited to:

- **Documentation** – Writing, editing, improving documents
- **Code and Technical** – Creating tools, fixes, or enhancements
- **Research** – Conducting research contributing to framework
- **Feedback** – Providing constructive feedback and review
- **Community** – Community building and support
- **Translation** – Translating materials to other languages
- **Governance** – Participating in decision-making and oversight

### Attribution Practices

#### In Code and Documentation
- Author names in file headers or comments
- Contributor lists in README or CONTRIBUTORS file
- Commit attribution in version control
- Acknowledgments in documentation

#### Public Recognition
- Credits section in release notes
- GitHub contributors page
- Community announcements
- Public acknowledgment in relevant forums

#### Levels of Recognition

**Major Contributors** – Significant, sustained contributions
- Listed prominently in documentation
- Mentioned in release notes
- Recognized in public communications

**Contributors** – Meaningful contributions
- Listed in contributors list
- Credited in relevant documentation
- Recognized in community forums

**Supporters** – Feedback, ideas, and support
- Thanked in acknowledgments
- Recognized through issues and discussions
- Credited as appropriate

## Citation and Source Standards

### Citing Ethical AI MY

When referencing this work, please use:

```
Ethical AI MY. (2026). Open reference for ethical AI development in Malaysia. 
Retrieved from https://github.com/AnuarRazii/ethical-ai-my
```

Bibliographic format:
```
Anuar Razii et al. (2026). Ethical AI MY – Reference Release v1.0. 
GitHub. https://github.com/AnuarRazii/ethical-ai-my
```

### Citing Specific Documents

For specific policy documents:

```
Ethical AI MY. (2026). "Ethics Framework." In Ethical AI MY: Open Reference 
for Ethical AI in Malaysia. Retrieved from https://github.com/AnuarRazii/ethical-ai-my/blob/main/ETHICS.md
```

### Derivative Works

If you create derivative works or adaptations:

- Clearly indicate the original source
- Specify what has been modified
- Maintain attribution to original authors
- Follow the CC BY 4.0 license requirements
- Share improvements back with community where appropriate

## Intellectual Property Recognition

### Open Content Licensing

Ethical AI MY is released under CC BY 4.0, which:

- Allows free use, modification, and distribution
- Requires attribution to original creators
- Does not impose restrictions on commercial use
- Enables broad adoption and adaptation

### Source Material Attribution

When incorporating source materials:

- Clearly identify the source
- Include original author attribution
- Follow source material licensing requirements
- Maintain links to original works
- Document modifications made

### Research and References

When building on research or ideas:

- Cite academic and industry sources
- Acknowledge research that informed the framework
- Link to relevant papers and resources
- Credit thought leaders and pioneers
- Support ongoing research efforts

## Community Contribution Process

### Contributing Your Work

1. **Identify contribution** – What will you contribute?
2. **Create or modify** – Develop your contribution
3. **Attribution** – Ensure proper attribution of sources
4. **Submission** – Submit through pull request
5. **Review** – Community review process
6. **Merge** – Contribution integrated
7. **Recognition** – Contributor recognized

### Recognition Upon Merge

- Author name in commit
- Added to contributors list
- Mentioned in relevant documentation
- Recognized in release notes if significant
- Thanked in community communications

## Historical Record Maintenance

### Version History

### Long-term Recognition

- Major contributors recognized in foundational sections
- Ongoing contributors listed annually
- Community acknowledged in acknowledgments section
- Partnerships and collaborations documented

## Recognition Guidelines

### For Individuals

- Use preferred name or pseudonym
- Include affiliation if desired
- Provide contact information if comfortable
- Specify areas of contribution

### For Organizations

- Organizational name and website
- Type of contribution (financial, technical, institutional support)
- Logo and description (if appropriate)
- Contact person for partnership

### For References and Sources

- Full citation information
- URL or DOI when available
- Date accessed for web sources
- License information if applicable

## Ethical Attribution

### Principles

- **Accuracy** – Attributions should be accurate and verifiable
- **Completeness** – All contributors are recognized
- **Timeliness** – Recognition provided promptly
- **Visibility** – Recognition is easily discoverable
- **Respect** – Recognition reflects contributor preferences

### Addressing Attribution Issues

If you believe attribution is incomplete or inaccurate:

1. Open an issue describing the concern
2. Provide evidence supporting the correction
3. Engage respectfully in discussion
4. Work toward fair resolution
5. Implement corrections promptly
